# aiR (r-copilot) Refined MVP Plan

## SECTION 1 — PRODUCT DEFINITION
**Definition:** aiR is a conversational R data science workbench that bridges natural language goals with a stateful R runtime, serving both as an autonomous analyst and a Socratic tutor.

---

## SECTION 2 — ARCHITECTURE DECISION
**Decision:** **Option A** (Backend-orchestrated Gemini + Vertex AI Search + Private R Service).

**Compelling Reason:** This provides the tightest control over the execution loop. Since the frontend must never call the R runtime directly and code must be reviewed before execution (Constraint #8), the Python backend acts as the strict transactional boundary.

---

## SECTION 3 — SYSTEM ARCHITECTURE & HARD DECISIONS

1.  **Control Plane (Python FastAPI):** The source of truth for sessions, policy (slider), and grounding. It coordinates all calls between the UI, Vertex AI, and the R runtime.
2.  **Execution Plane (Private R Plumber):** An isolated Cloud Run service with NO public ingress. It receives code, restores state from GCS, executes, and persists state back to GCS.
3.  **Grounded Tutoring (Vertex AI Search):** Used specifically for R documentation, package help, and statistical theory retrieval to ground the agent's teaching mode.
4.  **State Persistence:** Every execution is a stateless transaction at the container level. Workspace state (`.RData`) and object metadata are persisted to Cloud Storage after every successful run.
5.  **Security (Signed URLs):** No public buckets. Plots and artifacts are served to the frontend via time-limited signed URLs generated by the Python backend.
6.  **Authentication:** Service-to-service IAM authentication ensures only the Python Control Plane can invoke the R Runtime.
7.  **Data Flow:** UI -> Python (Auth/Policy) -> R Runtime (Execute/Persist) -> Python (Sign Artifacts) -> UI.

## SECTION 4 — API & RUNTIME CONTRACTS

### Runtime Contract (Python <-> R)
**Endpoint:** `POST /execute`
**Security:** IAM OIDC Authentication

**Request Schema (JSON):**
```json
{
  "session_id": "string",
  "code": "string",
  "persist_bucket": "string",
  "options": {
    "capture_plots": true,
    "return_objects": true
  }
}
```

**Response Schema (JSON):**
```json
{
  "status": "success | error",
  "stdout": "string",
  "stderr": "string",
  "plots": ["path/to/gcs/plot_1.png"],
  "environment": [
    {"name": "df", "type": "data.frame", "rows": 150, "cols": 5}
  ],
  "snapshot_uri": "gs://bucket/sessions/id/snapshots/v1.RData"
}
```

## SECTION 6 — DEPLOYMENT COMMANDS

### 1. Setup Infrastructure
```powershell
# Create Buckets
gsutil mb -p $PROJECT_ID -l us-central1 gs://$PROJECT_ID-artifacts
gsutil mb -p $PROJECT_ID -l us-central1 gs://$PROJECT_ID-sessions

# Create Artifact Registry
gcloud artifacts repositories create air-repo --repository-format=docker --location=us-central1
```

### 2. Deploy R Runtime (Private)
```powershell
gcloud run deploy air-r-runtime \
  --source ./r-service \
  --no-allow-unauthenticated \
  --region us-central1 \
  --memory 2Gi
```

### 3. Deploy Control Plane (Public)
```powershell
gcloud run deploy air-control-plane \
  --source ./api \
  --allow-unauthenticated \
  --region us-central1 \
  --set-env-vars R_RUNTIME_URL=[URL_FROM_STEP_2],SESSION_BUCKET=$PROJECT_ID-sessions
```
